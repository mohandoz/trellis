**⚠️ Warning — ci-gate empty-check bypass (REL-02):**
If the tagged commit has zero check-runs (e.g., commit never pushed to CI-triggering branch), `result` is empty string, `failed` resolves to empty, and ci-gate passes without having verified anything. Fix: add `[ -n "$result" ] || { echo "FAIL: no CI checks found for this commit"; exit 1; }`. Not a production blocker in normal workflow but exploitable with unusual push patterns.

**⚠️ Warning — ci-gate filter name mismatch (REL-02):**
Filter `select(.name != "Release")` targets the workflow name "Release" but GitHub check-run names are job IDs (e.g., "ci-gate", "release", "docker", "homebrew"). The filter is effectively a no-op. Harmless in practice since sibling jobs run after ci-gate, but the intent is unfulfilled.
