<objective>
Wave 1: build the `conjure adopt` command surface and the forward 5-step pipeline that turns the Phase 21 primitives into a complete, audited, snapshot-backed adoption run. This is the bulk of the user-facing behavior: `cmd_adopt` (thin dispatcher), `scripts/adopt.sh` (orchestrator), the `.conjure-adopt-state` step-completion schema with crash-durable atomic writes, the dirty-tree gate, the dry-run zero-writes guarantee, and the adoption report.

This plan deliberately STOPS before rollback, partial-run recovery, and the `--apply-step`/`--update-manifest` op-executor (Plan 03, Wave 2) — those build on the state schema and pipeline skeleton established here, and they share `scripts/adopt.sh` so they cannot run in parallel. The signal trap and write-state-before-step durability ARE in this plan because they are inseparable from the forward pipeline (SIGKILL recovery depends on durable state written during the live run).

Phase 22 is orchestration, not invention: ~90% of the work is already-built Phase 21 libs (`snapshot_create`, `inventory_scan`/`inventory_emit_manifest`, `log_init`/`log_step`, `mutate_*`, `caps.sh`) plus the `init-project.sh existing` and `audit-setup.sh` subprocesses. Do not reimplement them — wire them.

Purpose: satisfy criteria 1 (dry-run), 2 (live run), and the static half of 3 (dirty-tree/--force) from ROADMAP, plus SAFE-01/04/06/07 and ADOPT-01/02/04/05/06.
Output: `cli/conjure` (cmd_adopt + dispatch + usage), `scripts/adopt.sh` (new pipeline orchestrator).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI flags (untrusted) → adopt.sh env vars | User-supplied `--apply-step <id>` / target path cross into the worker; must not enable path traversal or arbitrary write |
| target repo filesystem → snapshot/scaffold | adopt.sh reads + mutates an arbitrary user repo; every target mutation must route through lib/mutate.sh (RESTR-02 chokepoint) except the deliberate raw-cp snapshot |
| .conjure-adopt-state JSON ← jq writes | State is read back to drive recovery/rollback; malformed/truncated state must fail loudly, never mis-drive a mutation |
| dry-run boundary → target root | Dry-run MUST write zero bytes under the target; any leaked write is a trust + correctness violation (ADOPT-02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-04 | Tampering | dry-run leaking a write under target (Pitfall 1) | mitigate | Dry-run writes the manifest to a `mktemp -d` dir OUTSIDE the target (D-11); test asserts `git status --porcelain "$sb"` empty + no manifest under target |
| T-22-05 | Elevation of Privilege | snapshot recursively copying prior backups into itself (Pitfall 3) | mitigate | Self-copy guard at the orchestrator: snapshot only when no prior state; exclude/move-aside `.conjure-adopt-backups` before raw cp; regression test for two consecutive adopts |
| T-22-06 | Tampering | bypassing the mutate chokepoint with raw cp/printf on target files | mitigate | All target writes go through `mutate_*`; only `snapshot_create` (raw cp -a) and `.conjure-adopt-state` (atomic jq>tmp+mv) are the deliberate, scoped exceptions — preserves RESTR-02 |
| T-22-07 | Repudiation | a mid-run kill leaving no record of what changed | mitigate | log_step appends to RESTRUCTURE-LOG.md per step (SAFE-07, durable) AND state.json records sha256 before/after per step BEFORE the mutation (SAFE-04, SIGKILL-survivable) |
| T-22-08 | Spoofing | dirty-tree gate fooled by untracked-only changes (Pitfall 5) | mitigate | Use `git status --porcelain` (catches tracked-modified AND untracked), not `git diff --quiet`; dirty + no --force → exit 2 |
| T-22-SC | Tampering | npm/pip/cargo installs | accept | Zero external packages installed this phase (RESEARCH Package Legitimacy Audit: none); pure shell orchestration of in-repo libs — no slopcheck checkpoint required |
</threat_model>
