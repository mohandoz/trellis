- Then: `send -h "conjure audit\r"` then `expect_prompt` then `sleep 2`
- Then: `send "exit\r"` then `expect eof`
- Use `$env(CAST_FILE)` to receive the cast file path from the shell environment

Run expect: `expect "$DEMO_DIR/demo.exp"` after exporting `CAST_FILE`

- `agg --speed 1.5 --idle-time-limit 2 --theme dracula "$CAST_FILE" "$GIF_FILE"`

- `printf '[record-demo] File size: %s bytes\n' "$(wc -c < "$ASSETS_DIR/demo.gif")"`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer shell → expect subprocess | expect spawns asciinema rec; the recorded shell inherits the developer's environment |
| expect-spawned shell → temp dir | all file I/O from the recorded commands lands in DEMO_DIR (isolated) |
| DEMO_DIR → .github/assets/ | GIF is copied out; cast file stays in temp dir and is deleted on EXIT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | record-demo.sh writes to $DEMO_DIR | mitigate | mktemp -d creates a fresh, user-owned temp dir each run; trap EXIT removes it unconditionally — no stale state from prior runs |
| T-05-02 | Information Disclosure | asciinema captures developer's terminal env inside recording | mitigate | script sets PS1='$ ' as first command inside the recording to normalize the prompt; recording runs in $DEMO_DIR (not $HOME) so real .claude/ dirs are never visible in the cast |
| T-05-03 | Denial of Service | demo.gif committed binary bloat | accept | agg --idle-time-limit 2 + --speed 1.5 bounds GIF size; size is printed by record-demo.sh for manual review; target < 5 MB per RESEARCH.md Pitfall 5 |
| T-05-04 | Tampering | asciinema/agg/expect are system tools run as current user | accept | all three are homebrew-core approved (Package Legitimacy Audit in RESEARCH.md); no npm/pip packages involved; no install in CI |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | no npm/pip/cargo packages installed in this phase; tools are OS package manager only (homebrew/apt) — no slopcheck required |
</threat_model>
