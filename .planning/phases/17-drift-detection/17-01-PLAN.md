  <action>
    Create scripts/check.sh with the following structure. Do NOT use declare -A or mapfile — bash 3.2 on macOS system bash (/bin/bash) does not support them. All membership tests use a newline-delimited temp file and grep -qF (per lib/merge.sh pattern).

    SHEBANG AND HEADER:
    - #!/usr/bin/env bash
    - Comment block: purpose, usage, exit codes (0=current, 1=drift, 2=bad args)
    - set -uo pipefail (matches update.sh; NO set -e — it masks exit code propagation)

    ENV VAR SETUP (top of script, before any logic):
    - TARGET="${1:-$(pwd)}"
    - CONJURE_HOME="${CONJURE_HOME:-$(cd "$(dirname "$0")/.." && pwd)}"
    - PORCELAIN="${CONJURE_PORCELAIN:-0}"

    TARGET VALIDATION:
    - Guard: [ -d "$TARGET" ] || { echo "✗ target is not a directory: $TARGET" >&2; exit 2; }
    - This prevents path traversal issues (per 17-RESEARCH.md security note)

    SHA256 HELPER (sha256_file function):
    - Per D-cross-platform from 17-CONTEXT.md: check `command -v sha256sum` first; if present use `sha256sum "$1" | awk '{print $1}'`; else fall back to `shasum -a 256 "$1" | awk '{print $1}'`
    - Use two-line local pattern to avoid SC2155: `local h; h="$(sha256sum "$1" | awk '{print $1}')"` — though CI excludes SC2155, follow the safe form

    MANIFEST BUILD (build_manifest function or inline — inline is simpler):
    - MANIFEST="$(mktemp)"
    - trap 'rm -f "$MANIFEST"' EXIT
    - Emit each relative harness path (one per line) into MANIFEST with printf '%s\n':
      - Root dotfiles: .editorconfig, .gitattributes, .claudeignore (3 entries)
      - Config: .claude/settings.json (1 entry — NOT .claude/settings.json.tmpl; the harness path strips .tmpl)
      - Hooks: loop `for hook in "$CONJURE_HOME"/templates/hooks-nodejs/*.mjs` — emit `.claude/hooks/$(basename "$hook")` — the *.mjs glob explicitly excludes README.md (per Pitfall 3)
      - Skills: loop `for skill_dir in "$CONJURE_HOME"/templates/skills/*/` — emit `.claude/skills/$(basename "$skill_dir")/SKILL.md`
      - Agents: loop `for agent in "$CONJURE_HOME"/templates/agents/*.md` — emit `.claude/agents/$(basename "$agent")`
    - Total should be 35 entries (3 + 1 + 6 + 19 + 6); exact count verified against 17-RESEARCH.md manifest section

    KIT-FILE CLASSIFIER (read from MANIFEST, classify each entry as M or R):
    - Variables: modified="" removed="" added="" (empty string accumulators)
    - `while IFS= read -r rel; do ... done < "$MANIFEST"`
    - Inside the loop, resolve the kit source file via case statement (critical — per Pitfall 2):
      - .claude/settings.json) kit_file="$CONJURE_HOME/templates/settings.json.tmpl" ;;
      - .claude/hooks/*) kit_file="$CONJURE_HOME/templates/hooks-nodejs/$(basename "$rel")" ;;
      - .claude/skills/*/SKILL.md) extract skill_name via two-step param expansion: skill_name="${rel#.claude/skills/}"; skill_name="${skill_name%/SKILL.md}"; kit_file="$CONJURE_HOME/templates/skills/$skill_name/SKILL.md" ;;
      - .claude/agents/*) kit_file="$CONJURE_HOME/templates/agents/$(basename "$rel")" ;;
      - .editorconfig|.gitattributes|.claudeignore) kit_file="$CONJURE_HOME/templates/$rel" ;;
      - *) continue ;;
    - harness_file="$TARGET/$rel"
    - If [ ! -f "$harness_file" ]: append "$rel\n" to removed
    - Else: compare hashes; if differ, append "$rel\n" to modified
    - Hash comparison: `kit_hash="$(sha256_file "$kit_file")"; harness_hash="$(sha256_file "$harness_file")"` — two-line form avoids subshell exit-code masking

    ADDED-FILE DETECTOR (find harness .claude/ files not in manifest):
    - `while IFS= read -r harness_file; do` ... `done < <(find "$TARGET/.claude" -type f 2>/dev/null | sort)`
    - rel="${harness_file#$TARGET/}"
    - case "$rel" in skip patterns (per Pitfall 6):
      - .claude/.conjure-*) continue ;;
      - .claude/COMPOUND-CANDIDATES.md) continue ;;
      - .claude/docs/*) continue ;;
    - `if ! grep -qF "$rel" "$MANIFEST" 2>/dev/null; then added="$added$rel\n"; fi`

    DRIFT FLAG:
    - `drift=0`
    - `[ -n "$modified" ] || [ -n "$removed" ] || [ -n "$added" ] && drift=1` (or use if/then form for shellcheck clarity)

    OUTPUT (PORCELAIN branch first):
    - If PORCELAIN=1:
      - For each variable (modified, removed, added): `printf '%b' "$var" | while IFS= read -r f; do [ -n "$f" ] && printf 'M %s\n' "$f"; done`
      - Use M for modified, R for removed, A for added (per D-porcelain in 17-CONTEXT.md)
    - If PORCELAIN=0:
      - If drift=0: echo "Harness is current."
      - If drift=1:
        - Count each category: `mod_count=$(printf '%b' "$modified" | grep -c '[^[:space:]]' || true)` (grep -c counts non-blank lines; use `|| true` so pipefail does not fire on zero-count)
        - Print "Drift detected: $total file(s) differ from upstream kit"
        - Print "Note: modified files may include user customizations"
        - Print blank line
        - For each non-empty category, print "Modified ($mod_count):" then indented entries
        - Repeat for Removed and Added

    EXIT:
    - `exit "$drift"` as the final line (0 or 1)

    SHELLCHECK COMPLIANCE:
    - No SC2044 (use while+read loop, not for f in $(find...))
    - No declare -A (use temp file)
    - No mapfile
    - No local -n
    - Quoting: always double-quote expansions
    - process substitution `< <(find ...)` is bash-specific but shellcheck accepts it with `#!/usr/bin/env bash`
  </action>
