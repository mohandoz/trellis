Replace with the following block (insert in the exact position of the removed lines):

  # --apply: real 3-way merge via lib/merge.sh (MERGE-01, D-01)
  source "$CONJURE_HOME/lib/mutate.sh" \
    || { echo "✗ Failed to load lib/mutate.sh — check CONJURE_HOME ($CONJURE_HOME)"; return 1; }
  source "$CONJURE_HOME/lib/merge.sh" \
    || { echo "✗ Failed to load lib/merge.sh — check CONJURE_HOME ($CONJURE_HOME)"; return 1; }

  # Read pinned version from project (already read into $pinned above)
  local snap_dir="$target/.claude/.conjure-templates-${pinned}"

  # D-01: abort if snapshot missing
  if [ ! -d "$snap_dir" ]; then
    echo "✗ No base snapshot for v${pinned}. Re-run 'conjure init' to write one, then update."
    return 1
  fi

  # Backup before any mutation (pattern from cmd_migrate)
  if [ -d "$target/.claude" ]; then
    local ts; ts="$(date +%Y%m%d-%H%M%S)"
    local backup="$target/.claude.backup-${ts}"
    echo "▸ Backing up existing .claude/ → $backup"
    cp -R "$target/.claude" "$backup"
  fi

  echo "▸ Applying 3-way merge: v${pinned} → v${CONJURE_VERSION}"

  # MERGE-04: generated files — take upstream unconditionally (no 3-way merge)
  if [ -f "$CONJURE_HOME/templates/settings.json.tmpl" ]; then
    mutate_cp "$CONJURE_HOME/templates/settings.json.tmpl" "$target/.claude/settings.json"
    echo "  ✓ settings.json taken from upstream (generated file)"
  fi

  # MERGE-01: user-owned files — 3-way merge via lib/merge.sh
  merge_user_files "$target" "$snap_dir" "$CONJURE_VERSION" "$pinned"
  local merge_rc=$?
  if [ "$merge_rc" -eq 2 ]; then
    echo "✗ Merge aborted due to git error — see messages above"
    return 2
  fi

  # D-06: report conflicts and exit 1; stamp version only on zero conflicts
  if [ "${CONJURE_MERGE_CONFLICT_COUNT:-0}" -gt 0 ]; then
    echo
    echo "✗ Conflicts in the following files:"
    for sf in $CONJURE_MERGE_CONFLICT_FILES; do
      echo "    $sf"
    done
    echo "  Resolve conflicts, delete the .conjure-conflict-* sidecars, then run:"
    echo "  echo '${CONJURE_VERSION}' > $target/.claude/.conjure-version"
    return 1
  fi

  # Clean: stamp new version
  mutate_write "$target/.claude/.conjure-version" "$CONJURE_VERSION"
  mutate_summary
  echo "✓ Updated to v${CONJURE_VERSION}"

Key implementation constraints:
- `pinned` is already set earlier in cmd_update (line 143: `pinned="$(cat "$target/.claude/.conjure-version")"`). Do not re-read it.
- Both `lib/mutate.sh` and `lib/merge.sh` must be sourced inside the --apply branch (cmd_update does not currently source either). Source mutate.sh FIRST (merge.sh depends on mutate_write).
- The `CONJURE_MERGE_CONFLICT_COUNT` and `CONJURE_MERGE_CONFLICT_FILES` module-level vars are set by lib/merge.sh's merge_user_files (which resets them at entry). Access them directly after the call.
- The backup `cp -R` is NOT guarded by DRY_RUN for now — cmd_update has no --dry-run flag. This matches the pattern in cmd_migrate line 108: `[ "$dryrun" = 0 ] && cp -R ...`. Since cmd_update has no dryrun local, always perform the backup (safe: creating a backup even in testing is harmless).
- MERGE-04: settings.json is the only generated file to take unconditionally (per D-03 and RESEARCH.md file classification table). .conjure-version is written by mutate_write at the end.
- The conflict summary loop iterates `$CONJURE_MERGE_CONFLICT_FILES` which is space-separated paths set by write_merge_sidecar in lib/merge.sh using the conditional-space expansion (no leading space). Use: `for sf in $CONJURE_MERGE_CONFLICT_FILES; do`.
