    - Per-sidecar prompt loop: derive current file path by stripping .conjure-conflict- prefix and reversing _ → / encoding; display sidecar filename and current file path; prompt "[k]eep / [a]pply / [e]dit / [s]kip: "
    - keep: call mutate_rm "$sidecar_path"; leave current file unchanged
    - apply: read sidecar content, call mutate_write "$current_file" "$content"; call mutate_rm "$sidecar_path"
    - edit: exec ${EDITOR:-vi} "$sidecar_path"; re-loop to re-prompt (do not auto-advance)
    - skip: print "  skipped" and move to next sidecar; leave both files in place
    - After all sidecars processed: recount remaining sidecars; if zero print "No conflicts remain"
    - Call mutate_summary at end

  <action>
    Write scripts/resolve.sh with shebang #!/usr/bin/env bash and set -uo pipefail. Derive CONJURE_HOME using: CONJURE_HOME="${CONJURE_HOME:-$(cd "$(dirname "$0")/.." && pwd)}". Source lib/mutate.sh: source "$CONJURE_HOME/lib/mutate.sh" with error exit on failure.

    TARGET="${1:-$(pwd)}"

    Non-interactive guard (per RESOLVE-01, D-non-interactive in CONTEXT.md): [ -t 0 ] || { echo "conjure resolve: stdin is not a TTY — interactive mode required" >&2; exit 2; }

    Sidecar discovery: collect sidecars via find "$TARGET" -name '.conjure-conflict-*' -type f into a temp file using mktemp; sort it (sort "$tmpfile" -o "$tmpfile"); trap 'rm -f "$tmpfile"' EXIT.

    All-clear check (per RESOLVE-02): if tmpfile is empty after sort, print "No conflicts remain" and exit 0.

    Main loop: while IFS= read -r sidecar_path from the tmpfile. For each sidecar:
      - sidecar_name=$(basename "$sidecar_path") — e.g. .conjure-conflict-skills_myskill_SKILL.md
      - encoded="${sidecar_name#.conjure-conflict-}" — strip prefix
      - rel="$(printf '%s' "$encoded" | tr '_' '/')" — decode back to relative path
      - current_file="$(dirname "$sidecar_path")/$rel" — resolve against sidecar's dir BUT sidecar_dir IS the same as current_file's dir, so: current_file="$(dirname "$sidecar_path")/../$rel" is WRONG. From merge.sh: sidecar_path = "${sidecar_dir}/${sidecar_name}" where sidecar_dir = dirname current_file. So: current_file = "$(dirname "$sidecar_path")/$rel" where $rel is the filename portion; but rel may include path separators. Actually since all sidecars sit in the same dir as their current file (dirname current_file = sidecar_dir), reconstruct: current_file = "$(dirname "$sidecar_path")/$rel". This is correct only when rel has no path separators — but rel IS the encoded relative path which may have slashes. Use the harness root (TARGET) instead: current_file = "$TARGET/$rel".
      - Print: "Sidecar: $sidecar_name" and "  Current file: $rel"
      - Inner prompt loop: while true; read -r -p "  [k]eep / [a]pply / [e]dit / [s]kip: " choice; case "$choice" in ...

    keep case (per RESOLVE-02): mutate_rm "$sidecar_path"; echo "  kept (sidecar removed)"; break

    apply case (per RESOLVE-02): read sidecar content into a variable: content="$(cat "$sidecar_path")"; mutate_write "$current_file" "$content"; mutate_rm "$sidecar_path"; echo "  applied (sidecar removed)"; break. NOTE: mutate_write takes content as a string arg, not via pipe (pipe = subshell = lost counter — see mutate.sh comment). Use content="$(cat "$sidecar_path")" before calling mutate_write.

    edit case: ${EDITOR:-vi} "$sidecar_path"; (no break — continues inner loop to re-prompt)

    skip case: echo "  skipped"; break

    *) echo "  Unknown choice; please enter k, a, e, or s"; (no break — re-prompt)

    After main loop: check remaining sidecars: remaining=$(find "$TARGET" -name '.conjure-conflict-*' -type f 2>/dev/null | wc -l | tr -d ' '); if [ "$remaining" -eq 0 ]; then echo "No conflicts remain"; fi

    mutate_summary

    Mark executable: chmod +x scripts/resolve.sh

    POSIX constraints: no declare -A, no mapfile, no local -n, bash 3.2 compatible (per CLAUDE.md). Use $() for subshells, not process substitution <(). No exit 1 for hard failures — use exit 2.

    Shellcheck: no SC2086 bare word expansions, no SC2155 combined declare+assign, quote all variable expansions.
  </action>
