  <behavior>
    Execution order (critical — all-clear must NOT require a TTY):
    1. source lib/mutate.sh
    2. Sidecar discovery: find "$TARGET" -name '.conjure-conflict-*' into sorted tmpfile
    3. All-clear check: if tmpfile is empty → print "No conflicts remain", exit 0 (NO TTY check here)
    4. Non-interactive guard: [ -t 0 ] || [ "${CONJURE_FORCE_INTERACTIVE:-0}" = "1" ] || exit 2
    5. Main prompt loop

    - Non-interactive guard fires only when sidecars ARE present and stdin is not a TTY and CONJURE_FORCE_INTERACTIVE is not 1; exits 2 with stderr "conjure resolve: stdin is not a TTY — interactive mode required"
    - CONJURE_FORCE_INTERACTIVE=1 bypasses the TTY check (testing escape hatch; unset in production)
    - Per-sidecar prompt loop: derive current file path by stripping .conjure-conflict- prefix and reversing _ → / encoding; display sidecar filename and current file path; prompt "[k]eep / [a]pply / [e]dit / [s]kip: "
    - keep: call mutate_rm "$sidecar_path"; leave current file unchanged
    - apply: read sidecar content, call mutate_write "$current_file" "$content"; call mutate_rm "$sidecar_path"
    - edit: exec ${EDITOR:-vi} "$sidecar_path"; re-loop to re-prompt (do not auto-advance)
    - skip: print "  skipped" and move to next sidecar; leave both files in place
    - After all sidecars processed: recount remaining sidecars; if zero print "No conflicts remain"
    - Call mutate_summary at end
    - All sidecars must come from sorted find output for deterministic ordering (per D-sidecar-discovery in CONTEXT.md)
  </behavior>
  <action>
    Write scripts/resolve.sh with shebang #!/usr/bin/env bash and set -uo pipefail. Derive CONJURE_HOME using: CONJURE_HOME="${CONJURE_HOME:-$(cd "$(dirname "$0")/.." && pwd)}". Source lib/mutate.sh: source "$CONJURE_HOME/lib/mutate.sh" with error exit on failure.

    TARGET="${1:-$(pwd)}"

    ORDERING IS CRITICAL — follow this exact sequence:

    Step 1 — Sidecar discovery (BEFORE any TTY check):
    Collect sidecars via find "$TARGET" -name '.conjure-conflict-*' -type f into a temp file using mktemp; sort it (sort "$tmpfile" -o "$tmpfile"); trap 'rm -f "$tmpfile"' EXIT.

    Step 2 — All-clear check (per RESOLVE-02, NO TTY required):
    if [ ! -s "$tmpfile" ]; then echo "No conflicts remain"; exit 0; fi

    Step 3 — Non-interactive guard (per RESOLVE-01, D-non-interactive in CONTEXT.md):
    [ -t 0 ] || [ "${CONJURE_FORCE_INTERACTIVE:-0}" = "1" ] || { echo "conjure resolve: stdin is not a TTY — interactive mode required" >&2; exit 2; }
    NOTE: CONJURE_FORCE_INTERACTIVE=1 is a test-only escape hatch that bypasses the TTY check when stdin is a pipe. It does not weaken the production path since the env var is unset by default.

    Step 4 — Main prompt loop: while IFS= read -r sidecar_path from the tmpfile. For each sidecar:
      - sidecar_name=$(basename "$sidecar_path") — e.g. .conjure-conflict-skills_myskill_SKILL.md
      - encoded="${sidecar_name#.conjure-conflict-}" — strip prefix
      - rel="$(printf '%s' "$encoded" | tr '_' '/')" — decode back to relative path
      - current_file="$TARGET/$rel" — resolve against harness root (TARGET), not sidecar dir
      - Print: "Sidecar: $sidecar_name" and "  Current file: $rel"
      - Inner prompt loop: while true; read -r -p "  [k]eep / [a]pply / [e]dit / [s]kip: " choice; case "$choice" in ...

    keep case (per RESOLVE-02): mutate_rm "$sidecar_path"; echo "  kept (sidecar removed)"; break

    apply case (per RESOLVE-02): read sidecar content into a variable: content="$(cat "$sidecar_path")"; mutate_write "$current_file" "$content"; mutate_rm "$sidecar_path"; echo "  applied (sidecar removed)"; break. NOTE: mutate_write takes content as a string arg, not via pipe (pipe = subshell = lost counter — see mutate.sh comment). Use content="$(cat "$sidecar_path")" before calling mutate_write.

    edit case: ${EDITOR:-vi} "$sidecar_path"; (no break — continues inner loop to re-prompt)

    skip case: echo "  skipped"; break

    *) echo "  Unknown choice; please enter k, a, e, or s"; (no break — re-prompt)

    After main loop: check remaining sidecars: remaining=$(find "$TARGET" -name '.conjure-conflict-*' -type f 2>/dev/null | wc -l | tr -d ' '); if [ "$remaining" -eq 0 ]; then echo "No conflicts remain"; fi

    mutate_summary

    Mark executable: chmod +x scripts/resolve.sh

    POSIX constraints: no declare -A, no mapfile, no local -n, bash 3.2 compatible (per CLAUDE.md). Use $() for subshells, not process substitution <(). No exit 1 for hard failures — use exit 2.

    Shellcheck: no SC2086 bare word expansions, no SC2155 combined declare+assign, quote all variable expansions.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sidecar path → shell | Filenames from find output used in variable expansions and passed to rm/write |
| sidecar content → current file | File content written to current_file via mutate_write — no shell exec, safe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | sidecar_path in loop | mitigate | All paths derived from find output with quoted expansions; no eval; no unquoted $() in rm args |
| T-18-02 | Elevation of Privilege | ${EDITOR:-vi} invocation | accept | Editor is user-controlled; this is the intended interaction surface; no privilege boundary crossed |
| T-18-03 | Denial of Service | find on large TARGET tree | accept | Sidecars are at most one per merged file; count is bounded by harness size (≤50 files) |
| T-18-04 | Tampering | CONJURE_FORCE_INTERACTIVE env var | accept | Test-only escape hatch; unset in production; no security boundary crossed since resolve only removes sidecar files in project tree |
| T-18-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan — pure bash, no new deps |
</threat_model>
