<objective>
Create the two overlay worker scripts: scripts/init-overlay.sh (clone + apply) and
scripts/refresh-overlay.sh (re-pull + re-apply). These scripts implement the core overlay
mechanics: shallow git clone into mktemp dir, copy overlay files into .claude/ via
mutate_cp (excluding .git/), write .conjure-org-overlay marker, cleanup temp dir.

Purpose: All overlay state machine lives in these two scripts. The CLI (Plan 02) is just
a thin dispatcher. Tests (Plan 03) invoke these scripts directly via CONJURE_HOME.

Output: scripts/init-overlay.sh and scripts/refresh-overlay.sh, shellcheck-clean, all
writes through lib/mutate.sh, no credential storage anywhere.
</objective>

  <action>
    Create scripts/init-overlay.sh as a standalone worker script (not sourced). Follow the publish-plugin.sh structural template exactly: shebang, comment block, set -euo pipefail, CONJURE_HOME self-resolution via cd dirname, source lib/mutate.sh, DRY_RUN default.

    Script signature: positional args OVERLAY_URL="${1:-}" TARGET="${2:-$(pwd)}".

    Exit code contract (document in header comment):
    - 0 = success
    - 1 = user-fixable error (empty URL, clone failure)
    - 2 = hard prerequisite failure (git not installed, lib/mutate.sh missing)

    Prerequisite checks: command -v git or exit 2; [ -f "$CONJURE_HOME/lib/mutate.sh" ] or exit 2; [ -z "$OVERLAY_URL" ] or exit 1 with usage hint.

    Core sequence (per D-07, D-08, Pitfall 1, Pitfall 2, Pitfall 4):
    1. CLONE_TMP="$(mktemp -d)"
    2. echo "▸ Cloning overlay: $OVERLAY_URL"
    3. git clone --depth 1 "$OVERLAY_URL" "$CLONE_TMP" 2>/dev/null — on failure: rm -rf temp, exit 1
    4. CLONE_SHA="$(git -C "$CLONE_TMP" rev-parse HEAD)"
    5. Copy files via process substitution (never pipe — Pitfall 2): while IFS= read -r item; do mutate_cp "$item" "$TARGET/.claude/"; done < <(find "$CLONE_TMP" -mindepth 1 -maxdepth 1 ! -name '.git')
    6. rm -rf "$CLONE_TMP"
    7. mutate_write "$TARGET/.claude/.conjure-org-overlay" "$(printf 'url=%s\nsha=%s' "$OVERLAY_URL" "$CLONE_SHA")" — marker written ONLY after copy succeeds (per Pitfall 4)
    8. mutate_summary
    9. exit 0

    Marker format (per D-09, RESEARCH.md Pattern 6): flat key=value, two lines: url=<url> then sha=<sha>. Use printf 'url=%s\nsha=%s' — NOT printf 'url=%s\nsha=%s\n' (no trailing newline, consistent with .conjure-version).

    Print "▸ Overlay applied from: $OVERLAY_URL" before mutate_summary (per RESEARCH.md open question — use mutate_summary for count; add a summary line for the overlay source).

    OVLY-05 invariant: do NOT include any variable or string containing 'password', 'credential', or 'token' anywhere in the file. Authentication is purely delegated to git's credential helper (D-09).

    The backup-before-mutate step is NOT needed here — init-overlay.sh applies on top of a freshly-inited .claude/; backup is only in refresh-overlay.sh.
  </action>

    Script signature: TARGET="${1:-$(pwd)}".

    Exit code contract:
    - 0 = success
    - 1 = user-fixable error (no marker configured, clone failure)
    - 2 = hard prerequisite failure (git not installed, lib/mutate.sh missing)

    Prerequisite checks: command -v git or exit 2; [ -f "$CONJURE_HOME/lib/mutate.sh" ] or exit 2.

    Core sequence (per D-03, D-04, D-07, D-08, Pitfall 5):
    1. OVERLAY_MARKER="$TARGET/.claude/.conjure-org-overlay"
    2. [ ! -f "$OVERLAY_MARKER" ] → echo "✗ No org overlay configured. Run conjure init --overlay <git-url> first." >&2; exit 1 (exact message per D-04)
    3. OVERLAY_URL="$(grep '^url=' "$OVERLAY_MARKER" | cut -d= -f2-)" (cut -d= -f2- preserves = in URLs)
    4. Backup step (Pitfall 5): [ "${DRY_RUN:-0}" = "0" ] && (if [ -d "$TARGET/.claude" ]; then ts="$(date +%Y%m%d-%H%M%S)"; backup="$TARGET/.claude.backup-${ts}"; echo "▸ Backing up existing .claude/ → $backup"; cp -R "$TARGET/.claude" "$backup" || { echo "✗ Backup failed — aborting" >&2; exit 1; }; fi)
    5. CLONE_TMP="$(mktemp -d)"
    6. echo "▸ Re-cloning overlay: $OVERLAY_URL"
    7. git clone --depth 1 "$OVERLAY_URL" "$CLONE_TMP" 2>/dev/null — on failure: rm -rf temp, exit 1
    8. NEW_SHA="$(git -C "$CLONE_TMP" rev-parse HEAD)"
    9. Copy via process substitution: while IFS= read -r item; do mutate_cp "$item" "$TARGET/.claude/"; done < <(find "$CLONE_TMP" -mindepth 1 -maxdepth 1 ! -name '.git')
    10. rm -rf "$CLONE_TMP"
    11. mutate_write "$TARGET/.claude/.conjure-org-overlay" "$(printf 'url=%s\nsha=%s' "$OVERLAY_URL" "$NEW_SHA")"
    12. echo "▸ Overlay refreshed from: $OVERLAY_URL"
    13. mutate_summary
    14. exit 0

    OVLY-05 invariant: do NOT include 'password', 'credential', or 'token' anywhere.

    Note on backup: The backup uses plain cp -R (not mutate_cp) because backing up .claude/ is a safety operation, not a mutation of .claude/ itself. Guard it with [ "${DRY_RUN:-0}" = "0" ] so backup is skipped in dry-run (per RESEARCH.md open question 2 and cli/conjure line 128 precedent).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → overlay URL argument | User-supplied git URL; passed to git clone without sanitization beyond non-empty check |
| overlay repo → .claude/ filesystem | Overlay repo content is copied wholesale; maintainers own all content |
| .claude/ → project git state | Overlay may write files tracked by the project's git repo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Tampering | overlay repo content → .claude/ | accept | D-02 design choice: org controls overlay; out of Conjure's threat model; org admins are trusted |
| T-12-02 | Information Disclosure | overlay URL logged in plain text when URL contains embedded credentials (e.g., https://token@host/repo) | mitigate | Do not echo OVERLAY_URL to stdout after reading it; if print needed, mask with sed 's|//[^@]*@|//***@|' or omit URL from progress output |
| T-12-03 | Tampering | .git/ injection from overlay clone into .claude/ | mitigate | find -mindepth 1 -maxdepth 1 ! -name '.git' copy pattern; verified in RESEARCH.md Pitfall 1 |
| T-12-04 | Elevation of Privilege | arbitrary file overwrite outside .claude/ | accept | mutate_cp uses cp -r; destination is always TARGET/.claude/ which is user-controlled; overlay maintainer cannot escape target dir via path traversal unless they control filenames with ../; this is an org-trust boundary |
| T-12-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this phase; all tools are POSIX + git builtins |
</threat_model>
