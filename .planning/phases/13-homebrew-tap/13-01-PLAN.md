    producing a hardcoded path in the wrapper — this is correct Homebrew practice.
    Do NOT call `brew --prefix` at runtime (per D-05).

    Current line 24:
      CONJURE_HOME="$(cd "$(dirname "$0")/.." && pwd)"

    Replace with (per D-03):
      CONJURE_HOME="${CONJURE_HOME:-$(cd "$(dirname "$0")/.." && pwd)}"

    The change wraps the existing subshell in a `${VAR:-default}` conditional. When
    CONJURE_HOME is already set in the environment (as the Homebrew wrapper will set it),
    the subshell never runs. When it is not set (direct invocation from a dev checkout),
    the behavior is identical to before.

    Line 25 already reads `$CONJURE_HOME/VERSION` — no change needed there.

    After editing, run shellcheck on cli/conjure to confirm no regressions. If shellcheck
    reports SC2155 (declare and assign separately) on the new line, add
    `# shellcheck disable=SC2155` on the line immediately above — but only if shellcheck
    actually flags it. The conditional assignment pattern is generally shellcheck-clean
    because it does not use local or declare.

    Verify the edit is exactly one line changed — do not touch any other line.
