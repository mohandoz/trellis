    The section iterates all 9 green fixtures via the `[^_]*/` glob (same glob that excludes `_broken`).

    Section shape (insert verbatim, preserving whitespace):

      echo
      echo "▸ Dry-run byte-identical snapshot (TEST-05)"
      for fx in "$CONJURE_HOME/tests/fixtures"/[^_]*/; do
        prof=$(basename "$fx")
        DRY_ORIG="$(mktemp -d)"
        DRY_SNAP="$(mktemp -d)"
        cp -r "$fx/." "$DRY_ORIG/"
        cp -r "$fx/." "$DRY_SNAP/"
        CONJURE_HOME="$CONJURE_HOME" cli/conjure init --dry-run "$DRY_SNAP" >/dev/null 2>&1 || true
        if diff -r "$DRY_SNAP" "$DRY_ORIG" >/dev/null 2>&1; then
          pass "dry-run snapshot identical: $prof"
        else
          fail "dry-run mutated tree: $prof"
          diff -r "$DRY_SNAP" "$DRY_ORIG" | head -10
        fi
        rm -rf "$DRY_ORIG" "$DRY_SNAP"
      done

    Critical constraints (from RESEARCH.md and PATTERNS.md):

    1. Do NOT call `sandbox_setup` in this section. `sandbox_setup` registers `trap 'rm -rf "$SANDBOX_DIR"'
       EXIT` (sandbox.sh line 35) which replaces bash's EXIT trap, clobbering any trap registered for
       DRY_ORIG/DRY_SNAP. It also overrides HOME/PATH — exported HOME would contaminate the diff since
       both DRY_ORIG and DRY_SNAP are copied from the same fixture but with different HOME values applied.
       Use plain `mktemp -d` instead (RESEARCH.md Pitfall 1, PATTERNS.md Pattern 2 Key Constraints).

    2. `rm -rf "$DRY_ORIG" "$DRY_SNAP"` must appear at the end of each loop iteration — explicit cleanup
       per iteration (not only via EXIT trap) ensures temp dirs are removed even when the next loop
       iteration calls `sandbox_setup` (which would replace the EXIT trap).

    3. Use `CONJURE_HOME="$CONJURE_HOME" cli/conjure init --dry-run "$DRY_SNAP"` — the
       CONJURE_HOME prefix ensures `lib/mutate.sh` and other sourced scripts resolve correctly.
       Matches the pattern at tests/run.sh line 195 (existing dry-run enforcement section).

    4. `|| true` after the conjure call — dry-run may exit non-zero if preflight fails in sandbox;
       the real assertion is the diff, not the conjure exit code.

    5. `diff -r` without `--brief` so failure output is diagnostic (shows which file changed).
       `| head -10` limits output on failure so the test log stays readable.

    6. Per D-06, this tests re-init idempotence — the fixtures already have `.claude/` (they were
       committed by Phase 3). Running `conjure init --dry-run` against an already-initialized fixture
       tests the real-world case of a second dry-run invocation.
  </action>

      printf '#!/usr/bin/env bash\nexit 1\n' > "$FM_DIR/.claude/hooks/bad-gate.sh"
      if grep -qE '^exit 1$' "$FM_DIR/.claude/hooks/bad-gate.sh"; then
        pass "FM: hook exit 1 detectable via grep"
      else
        fail "FM: hook exit 1 NOT found"
      fi
      rm -rf "$FM_DIR"

      # FM-3: .conjure-version mismatch — conjure update detects this
      # NOTE: audit-setup.sh does NOT check .conjure-version — use cli/conjure update (Finding F-01)
      # NOTE: version file must be at .claude/.conjure-version (not root level — Pitfall 5)

      if printf '%s\n' "$FM_OUT" | grep -q "pinned to" && \
         ! printf '%s\n' "$FM_OUT" | grep -q "Up to date"; then
        pass "FM: version mismatch detected by conjure update"
      else
        fail "FM: version mismatch NOT detected"
      fi
      rm -rf "$FM_DIR"

    - `CONJURE_HOME="$CONJURE_HOME" cli/conjure update "$FM_DIR"` — CONJURE_HOME prefix needed

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-32 | Tampering | FM-2: script injection via printf content into hook file | accept | Content is a hardcoded literal: `printf '#!/usr/bin/env bash\nexit 1\n'`. No external variable interpolation. The file is then read with `grep -qE` (not sourced/executed). |
| T-04-33 | Tampering | FM blocks leave temp dirs on failure | mitigate | `rm -rf "$FM_DIR"` is explicit at end of each FM block. Even if bash exits mid-block, the EXIT trap registered by the most recent `sandbox_setup` call in the EXPECT loop above will clean SANDBOX_DIR but NOT FM_DIR. Belt-and-suspenders: explicit rm + note that FM dirs are under /tmp (OS reaps on reboot at worst). |
| T-04-34 | Tampering | FM-1 size-cap test: adversarial EXPECT always-match pattern | accept | FM-1 asserts `HARD CAP exceeded` — this is a specific substring, not a regex with always-match properties. The string cannot match a passing fixture (which produces `PASS:` output). |
