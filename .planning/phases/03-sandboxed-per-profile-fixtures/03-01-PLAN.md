  <action>
    Create the directory `tests/lib/` and write `tests/lib/sandbox.sh` as a sourced bash library
    (no shebang line — it is source'd, not executed directly, matching the lib/mutate.sh contract).

    The file must open with a comment block (per D-04) documenting the public interface:
      - Source path: `source "$CONJURE_HOME/tests/lib/sandbox.sh"`
      - Public function: `sandbox_setup <fixture_dir>`
      - Output variable: `SANDBOX_DIR` (global, set by the function)
      - Env vars exported: `HOME`, `XDG_CONFIG_HOME`, `CLAUDE_CONFIG_DIR`, `PATH`
      - Callers MUST register their own trap: `trap 'rm -rf "$SANDBOX_DIR"' EXIT`
      - `CONJURE_HOME` is intentionally NOT overridden (per D-05, Pitfall 5)

    Implement `sandbox_setup()` with these exact behaviors (per D-04, D-05, D-06):
      1. Declare `local fixture_dir="$1"` (POSIX-safe scoping)
      2. Set global `SANDBOX_DIR="$(mktemp -d)"`
      3. Copy fixture: `cp -r "$fixture_dir/." "$SANDBOX_DIR/"` (the trailing `/. ` copies contents not
         the dir itself — same convention as mutate_cp in lib/mutate.sh)
      4. Export `HOME="$SANDBOX_DIR"`
      5. Export `XDG_CONFIG_HOME="$SANDBOX_DIR"`
      6. Export `CLAUDE_CONFIG_DIR="$SANDBOX_DIR"`
      7. Export `PATH="$CONJURE_HOME/cli:/usr/local/bin:/usr/bin:/bin"` (restricted: keeps system tools,
         adds CONJURE_HOME/cli for conjure command, strips developer-local tool paths)
      8. Do NOT export `CONJURE_HOME` — callers inherit the real kit location

    Do NOT add `set -euo pipefail` at the file top (this would affect caller shell options).
    Do NOT add a shebang line.
    Use `printf` not `echo` for any output (project-wide printf convention).
  </action>

    Create `scripts/regen-fixtures.sh` as an executable bash script with these behaviors:

    HEADER: `#!/usr/bin/env bash` + `set -euo pipefail` + `CONJURE_HOME` resolved via
    `"$(cd "$(dirname "$0")/.." && pwd)"`. Define `FIXTURES_DIR="$CONJURE_HOME/tests/fixtures"`.

    ARGUMENT PARSING (per Claude's discretion to add --profile flag, A5 assumption): Parse an
    optional `--profile <p>` flag that sets `PROFILE_FILTER`. If no flag is given, `PROFILE_FILTER`
    is empty and all 9 profiles are regenerated.

    Define `PROFILES` as a space-separated string or positional list of all 9 profile names
    (ts-next java-spring rust-axum go-gin python-fastapi node-nest monorepo polyglot data-science).
    When `PROFILE_FILTER` is set, only that one profile runs.

    Define helper `_write_manifest <profile> <seed_dir>`:
      - ts-next, node-nest, monorepo, polyglot: write `{"name":"fixture","version":"0.0.0"}` to
        `$seed/package.json` using printf
      - java-spring: write minimal `<project>` pom.xml stub using printf (single-line XML element
        with modelVersion, groupId, artifactId, version — see RESEARCH.md Code Examples)
      - rust-axum: write `[package]` Cargo.toml stub (name, version, edition = "2024") using printf
      - go-gin: write `module fixture\n\ngo 1.22\n` to `$seed/go.mod` using printf
      - python-fastapi, data-science: write `fastapi>=0.110\n` to `$seed/requirements.txt` using printf
      - monorepo ONLY: after writing package.json, also `mkdir -p "$seed/packages/api"` (Critical
        Finding 6 — without this, monorepo apply.sh exits 1 and skips fragment append)

    Define helper `_write_seed_claude <seed_dir>` (Critical Finding 1 — init does NOT create CLAUDE.md):
      Write a seed CLAUDE.md to `$seed/CLAUDE.md` using printf. Content must include:
        Line 1: `# GENERATED — do not edit directly; run scripts/regen-fixtures.sh` (per D-12)
        Followed by `## Project`, `Fixture project.`, `## Conventions`, `None.` and enough blank lines
        to reach ~22 lines total (verified: 22-line seed + profile fragment = 30–45 lines, well under
        100-line cap for all 9 profiles per RESEARCH.md Finding 7).

    Define `regen_profile <profile>`:
      1. Print `[regen] $p` (per CONTEXT.md Specific Ideas — consistent with [dry-run] prefix)
      2. Create seed: `local seed; seed="$(mktemp -d)"`
      3. Register RETURN trap: `trap 'rm -rf "$seed"' RETURN` (scoped to function so each profile
         cleans up its own seed dir)
      4. Call `_write_manifest "$p" "$seed"`
      5. Call `_write_seed_claude "$seed"`
      6. Run init: `CONJURE_HOME="$CONJURE_HOME" "$CONJURE_HOME/cli/conjure" init --profile="$p" "$seed" >/dev/null`
         (suppress stdout — progress output is verbose; errors still propagate via set -e)
      7. Idempotent overwrite: `rm -rf "${FIXTURES_DIR:?}/$p"` then `cp -r "$seed/." "$FIXTURES_DIR/$p/"`
         (use `${FIXTURES_DIR:?}` safe expansion to prevent accidental `rm -rf /` if var unset)
      8. Self-validate (per RESEARCH.md open question 1): run `bash "$CONJURE_HOME/scripts/audit-setup.sh"
         "$FIXTURES_DIR/$p" >/dev/null 2>&1` and if it exits non-zero print `[regen] WARN: $p fixture
         fails audit — check profile output` then `exit 1`
      9. Print `[regen] $p done`

    MAIN LOOP: Iterate `$PROFILES`; if `PROFILE_FILTER` is set, only call `regen_profile` for
    the matching profile; otherwise call it for all 9.

    Make the script executable: `chmod +x scripts/regen-fixtures.sh`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sandbox env → real $HOME | sandbox_setup exports HOME/XDG_CONFIG_HOME/CLAUDE_CONFIG_DIR — if not set correctly, audit invocations could read developer credentials |
| regen seed dir → committed fixtures | regen-fixtures.sh uses rm -rf to overwrite fixtures — if FIXTURES_DIR is unset, this becomes rm -rf / |
| CLI invocation in regen | conjure init in regen runs with CONJURE_HOME set to real kit — must not be re-pointed to seed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `sandbox_setup` — HOME export | mitigate | Export HOME/XDG_CONFIG_HOME/CLAUDE_CONFIG_DIR to $SANDBOX_DIR; do NOT override CONJURE_HOME so CLI resolves real kit paths |
| T-03-02 | Tampering | `regen_profile` — rm -rf $FIXTURES_DIR/$p | mitigate | Use `${FIXTURES_DIR:?}/$p` safe-expansion; set -euo pipefail exits before rm if var is empty |
| T-03-03 | Denial of Service | `sandbox_setup` — temp dir leak on error | mitigate | Document that callers MUST register `trap 'rm -rf "$SANDBOX_DIR"' EXIT`; sandbox_setup itself does not register a trap (setup-only form per D-05) |
| T-03-04 | Information Disclosure | Fixture audit reads developer $HOME | mitigate | PATH restriction in sandbox_setup (`$CONJURE_HOME/cli:/usr/local/bin:/usr/bin:/bin`) prevents spurious tool detection; HOME override prevents accidental ~/.claude reads |
| T-03-SC | Tampering | No npm/pip/cargo installs | accept | No new package-manager installs in this plan; no slopcheck needed |
</threat_model>
