<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sandbox env → test assertions | If sandbox_setup doesn't correctly isolate HOME, audit could read developer $HOME and produce non-deterministic results |
| EXPECT grep patterns → actual audit output | If the pattern doesn't match the exact audit message, TEST-04 assertion always passes vacuously (grep -qE "" always true) |
| tests/run.sh trap scope | If EXIT trap accumulates across loop iterations without reset, earlier $SANDBOX_DIR values persist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | sandbox isolation — real $HOME bleeds into audit | mitigate | sandbox_setup exports HOME=$SANDBOX_DIR; audit-setup.sh is env-isolated (RESEARCH.md Finding 3); belt-and-suspenders |
| T-03-10 | Spoofing | _broken EXPECT pattern matches vacuously (empty pattern) | mitigate | EXPECT reader skips blank lines and comment lines; EXPECT contains at least one concrete pattern `HARD CAP exceeded` |
| T-03-11 | Denial of Service | temp dir leak from fixture loop if trap fires early | mitigate | Trap registered per-iteration with current $SANDBOX_DIR value; EXIT trap fires at process end cleaning all registered paths |
| T-03-12 | Tampering | _broken CLAUDE.md has only 105 lines (WARN not ERR) | mitigate | Accept criteria asserts `wc -l` ≥ 201 AND audit exit code = 2 (not 1); Pitfall 2 documented in RESEARCH.md |
| T-03-13 | Information Disclosure | audit invoked via cli/conjure (runs preflight with dev PATH) | mitigate | Accept criteria asserts the new run.sh sections do NOT contain `cli/conjure audit`; use `bash scripts/audit-setup.sh` directly |
| T-03-SC | Tampering | No npm/pip/cargo installs | accept | No new package-manager installs; no slopcheck needed |
</threat_model>
