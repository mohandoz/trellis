<objective>
Extract the inline cmd_preflight() function from cli/conjure into a standalone scripts/preflight.sh
with proper OS detection, required/optional dep split, per-OS fix-it lines, and non-zero exit on
required-dep failure. Wire preflight as a user-facing subcommand, as the first gate in cmd_audit()
(per D-08), and add tests/run.sh verification.

Purpose: SAFE-04 — pre-flight must report each missing dep with a copy-pasteable OS-detected fix-it
and never auto-install. The existing cmd_preflight() never blocks (always returns 0), always emits
brew regardless of OS, and treats git/jq/rg identically. This plan fixes all three defects.
D-08 further requires that `conjure audit` calls preflight as its first step so audit fails fast
when required deps are missing.

Output:
- scripts/preflight.sh: new standalone script, self-contained, POSIX bash 3.2+
- cli/conjure: cmd_preflight() replaced with stub-delegate; preflight added to dispatch and usage();
  cmd_audit() gains `cmd_preflight || return 1` as its first executable line (D-08)
- tests/run.sh: preflight section with 4 assertions (smoke, block-on-required, warn-on-optional, fix-it output)
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell environment → scripts/preflight.sh | $PATH, $OSTYPE, uname output are environment-supplied — a hostile PATH could cause wrong dep detection |
| scripts/preflight.sh → stdout | Fix-it output goes to terminal; contains only static install-manager strings, no user input echoed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | $PATH manipulation in test env | accept | PATH manipulation is intentional in tests (node-strip); preflight only reads PATH to call command -v, does not write it |
| T-01-02 | Denial of Service | preflight.sh never exits (hangs on uname) | mitigate | use `uname -s 2>/dev/null || echo unknown` fallback so a missing uname degrades to "unknown" OS without hanging |
| T-01-03 | Elevation of Privilege | preflight.sh auto-installs a dep | mitigate | D-12: script contains ONLY printf/echo for fix-it lines, zero package-manager exec calls; verified by grep in tests/run.sh |
| T-01-04 | Spoofing | $OSTYPE unset, _detect_os returns wrong OS | accept | uname -s is primary; $OSTYPE is fallback only for windows-gitbash case; unknown OS falls through gracefully with a generic URL |
| T-01-SC | Tampering | npm/pip/cargo installs during implementation | mitigate | Phase 1 installs zero new packages; Package Legitimacy Audit confirms this (RESEARCH.md §Package Legitimacy Audit: "not applicable") |
